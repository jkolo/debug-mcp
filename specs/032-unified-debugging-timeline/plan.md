# Implementation Plan: Unified Debugging Timeline

**Branch**: `032-unified-debugging-timeline` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/032-unified-debugging-timeline/spec.md`

## Summary

Add a `debugger://timeline` MCP resource and a `timeline_query` tool that expose a chronological log of all significant debugging events from the current session — breakpoint/tracepoint hits, exceptions (first-chance and user-unhandled), module loads, thread lifecycle, stdout/stderr output, and session start/end markers. A new `TimelineStore` singleton subscribes to existing `IProcessDebugger` events, two new thread lifecycle events (`ThreadCreated`/`ThreadExited`), and a new `OutputReceived` event on `ProcessIoManager`. Events are stored in a capped in-memory circular buffer (10,000 events). Agents gain cross-modality event correlation with a single resource read instead of multiple tool calls.

## Technical Context

**Language/Version**: C# 13 / .NET 10.0

**Primary Dependencies**: ModelContextProtocol 1.3.0, ClrDebug 0.3.4, System.Collections.Concurrent (in-box)

**Storage**: In-memory only (`ConcurrentQueue<TimelineEvent>`, max 10,000 entries; cleared per session)

**Testing**: xUnit + FluentAssertions + Moq — unit tests for `TimelineStore` event subscription and filtering; contract test for `timeline_query` tool annotations

**Target Platform**: Linux/macOS/Windows (x64 + ARM64) — no platform-specific code

**Project Type**: MCP server / .NET global tool

**Performance Goals**: Event recording ≤ 100ms latency; `debugger://timeline` resource read ≤ 500ms for sessions with up to 1,000 events (SC-002, SC-003)

**Constraints**: In-memory cap at 10,000 events; no disk persistence; cleared on new session

**Scale/Scope**: Single in-process session; 1 timeline per server process

## Implementation Phases

### Phase 1 — Models

Create all 5 model files under `DebugMcp/Models/Timeline/`. No production code changes yet.

- `TimelineEventType` enum (11 values)
- `TimelineEventPayload` abstract record + 9 concrete subtypes (SessionStarted/Ended, BreakpointHit, TracepointHit, Exception, ModuleLoaded, ThreadStarted/Exited, Output)
- `TimelineEvent` positional record
- `TimelineFilter` positional record
- `TimelineResponse` positional record (read DTO)

### Phase 2 — New Event Hooks (IProcessDebugger + ProcessIoManager)

**IProcessDebugger additions** (2 new events):

```csharp
event EventHandler<ThreadCreatedEventArgs>? ThreadCreated;
event EventHandler<ThreadExitedEventArgs>? ThreadExited;
```

**ProcessDebugger** — wire them in existing `OnCreateThread` / `OnExitThread` callbacks (after auto-continue logic):

```csharp
callback.OnCreateThread += (sender, e) =>
{
    if (ShouldAutoContinue()) e.Controller.Continue(false);
    ThreadCreated?.Invoke(this, new ThreadCreatedEventArgs((int)e.Thread.Id));  // add
};
```

**ProcessIoManager** — add `OutputReceived` event; fire from `PumpStreamAsync` after each buffered line:

```csharp
public event Action<string, string>? OutputReceived;  // (content, stream)
// in PumpStreamAsync, after appending to buffer:
OutputReceived?.Invoke(truncatedLine, streamName);
```

### Phase 3 — TimelineStore (ITimelineStore + TimelineStore)

```csharp
public interface ITimelineStore
{
    void Record(TimelineEvent e);
    TimelineResponse GetAll();
    TimelineResponse GetFiltered(TimelineFilter filter);
    void Clear();
    int TotalRecorded { get; }
    int EventsDropped { get; }
}
```

`TimelineStore` is a singleton that receives its event sources via constructor injection:
- `IProcessDebugger` — subscribes to `StateChanged`, `BreakpointHit`, `ModuleLoaded`, `ExceptionHit`, `ThreadCreated`, `ThreadExited`
- `ProcessIoManager` — subscribes to `OutputReceived`

Event → TimelineEvent mapping:

| Source event | TimelineEventType | Key payload fields |
|---|---|---|
| `StateChanged(Running)` first time | `SessionStarted` | session_type, pid |
| `StateChanged(Disconnected)` | `SessionEnded` | reason |
| `BreakpointHit` (tracepoint=false) | `BreakpointHit` | breakpoint_id, file, line |
| `BreakpointHit` (tracepoint=true) | `TracepointHit` | tracepoint_id, file, line |
| `ExceptionHit` (first-chance) | `ExceptionFirstChance` | type, message, thread_id |
| `ExceptionHit` (user-unhandled) | `ExceptionUserUnhandled` | type, message, thread_id |
| `ModuleLoaded` | `ModuleLoaded` | module_name, path, has_symbols |
| `ThreadCreated` | `ThreadStarted` | thread_id (name=null) |
| `ThreadExited` | `ThreadExited` | thread_id (name=null) |
| `OutputReceived` ("stdout") | `StdoutWritten` | content, truncated |
| `OutputReceived` ("stderr") | `StderrWritten` | content, truncated |

`StateChanged(Disconnected)` also calls `Clear()` after recording `SessionEnded` (FR-011, FR-014).

**Cap and eviction**:

```csharp
private void Append(TimelineEvent e)
{
    if (_events.Count >= MaxCapacity)
    {
        _events.TryDequeue(out _);
        Interlocked.Increment(ref _eventsDropped);
    }
    _events.Enqueue(e);
    Interlocked.Increment(ref _totalRecorded);
}
```

### Phase 4 — Resource and Tool

**`debugger://timeline` resource** — add method to `DebuggerResourceProvider`:

```csharp
[McpServerResource(UriTemplate = "debugger://timeline", Name = "Debugging Timeline", MimeType = "application/json")]
public string GetTimelineJson()
{
    var response = _timelineStore.GetAll();
    return JsonSerializer.Serialize(response, JsonOptions);
}
```

Returns empty timeline (`events: []`) when no session is active (no exception thrown).

**`timeline_query` tool**:

```csharp
[McpServerToolType]
public sealed class TimelineQueryTool
{
    [McpServerTool(Name = "timeline_query", Title = "Query Timeline",
        ReadOnly = true, Destructive = false, Idempotent = true, OpenWorld = false)]
    [Description("Filter the debugging timeline by event type, thread, or cursor. Returns a subset of recorded events...")]
    public string TimelineQueryAsync(
        string? eventTypes = null,   // JSON array of strings, null = all
        int? threadId = null,
        int? fromEventId = null,
        int maxEvents = 200)
    {
        var filter = new TimelineFilter(...);
        return JsonSerializer.Serialize(_store.GetFiltered(filter), ...);
    }
}
```

### Phase 5 — DI Registration (Program.cs)

```csharp
// Register TimelineStore as singleton
builder.Services.AddSingleton<ITimelineStore>(sp =>
{
    var debugger = sp.GetService<IProcessDebugger>();
    var ioManager = sp.GetService<ProcessIoManager>();
    var logger = sp.GetRequiredService<ILogger<TimelineStore>>();
    return new TimelineStore(debugger, ioManager, logger);
});
```

`DebuggerResourceProvider` receives `ITimelineStore` as optional constructor parameter (same pattern as `ISnapshotStore`).
